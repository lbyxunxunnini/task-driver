# 任务驱动

任务驱动是一个低认知负担的复杂任务推进 skill。它不默认生成中间记录，而是用一条固定链路把任务推进到可验收结果。

## 30 秒选择器

使用任务驱动：

- 目标、范围或验收不清楚。
- 涉及多个文件、模块或外部系统。
- 涉及权限、费用、发布、分支、提交、合并、删除或覆盖重要文件。
- 用户要求先计划、先澄清或完整推进。

不使用任务驱动：

- 简单问答。
- 只需要执行一个安全命令。
- 单文件低风险小改动，且用户明确要求直接改。

谨慎使用：

- 用户要求立即执行，但任务存在风险动作时，先确认授权和验收标准。
- 任务看似很小，但失败会影响生产、数据、费用或不可逆状态时，按任务驱动推进。

## 核心链路

```text
仔细询问
-> 生成计划
-> 讨论计划
-> 生成验收标准
-> 讨论验收标准
-> 创建目标并执行
-> 自检未过则打回修复，最多 3 次
-> 无法修复则给出结论
```

## 解决的问题

- 澄清阶段问得太浅。
- 计划和验收标准没有真正讨论。
- 模型自制虚假验收标准。
- 执行前写太多用户看不懂的中间文档。
- 用户可见输出混入机器化流程名、状态名和内部名词。
- 执行阶段越权新建分支、提交、合并或消耗外部资源。
- 执行后没有围绕目标形成“自检、打回、成功”的闭环。
- 长执行缺少单一可恢复摘要，容易在上下文压缩后丢状态。

## 使用方式

当用户消息以以下任一形式开头时，进入任务驱动：

```text
tdr- ...
task-driver ...
/task-driver ...
```

示例：

```text
tdr- 帮我把阶段2接入生图能力重新梳理并实现
```

## 行为原则

- 用户可见输出全部使用中文自然语言。
- 每次澄清最多问一个关键问题。
- 计划只在对话里展示，默认不写文件。
- 验收标准必须单独生成、单独讨论、单独确认。
- 用户没有确认最终验收标准前，不进入执行。
- 最终验收标准确认后必须创建目标或维护轻量目标。
- 任何验收项未达成都必须打回修复，全部达成后才能完成目标。
- 高风险任务必须说明复核结果或用户复核建议。
- 一个项目的一个任务最多只维护一份 `.task-driver/{任务标题}-{时间戳}-计划摘要.md`。
- 大任务、超大任务或长执行由模型自主判断必须写摘要，普通任务默认不写。
- 辅助测试不能替代用户验收标准。
- 不自动新建分支、提交、推送或合并。
- 不自动真实调用外部服务或消耗额度。
- 未达成验收标准时自动自检修复，最多 3 次。
- 3 次仍无法达成时停止盲修，给出明确结论。

## 最短示例

```text
用户：tdr- 修复登录页按钮错位

助手：我先确认一个关键点：只修登录按钮错位，还是顺带统一登录页所有按钮样式？

用户：只修错位

助手：计划草案：

| 序号 | 要做什么 | 范围 | 状态 |
|---|---|---|---|
| 1 | 读取登录页和按钮样式 | 只读 | 草案 |
| 2 | 修复按钮错位 | 登录页相关样式 | 草案 |
| 3 | 做样式检查 | 桌面和移动宽度 | 草案 |

请确认或修改计划。

用户：确认

助手：验收标准草案：

| 序号 | 验收标准 | 检查方式 | 状态 |
|---|---|---|---|
| 1 | 登录按钮在桌面宽度不再错位 | 样式检查 | 草案 |
| 2 | 登录按钮在移动宽度不再溢出 | 样式检查 | 草案 |
| 3 | 其他按钮样式不被改动 | 差异检查 | 草案 |

请修改、删除或确认这些验收标准。

用户：确认

助手：我创建目标并开始执行，目标是满足以上 3 条验收标准。
```

## 权限边界

以下动作必须先获得用户明确授权：

- 新建分支。
- 提交、推送、合并。
- 删除、覆盖或移动重要文件。
- 发布、部署或改生产配置。
- 真实调用外部接口、消耗额度或产生费用。

任务完成后如果处在临时分支上，必须询问用户是否合并、保留、提交、推送或放弃。

## 轻量计划摘要

普通任务不写摘要。大任务、超大任务、长执行、预计无法一轮完成、验收标准很多、计划步骤很多、涉及多个模块或外部系统、有风险动作、发生自检打回或可能上下文丢失时，模型必须维护一份摘要：

```text
.task-driver/{任务标题}-{YYYYMMDDHHmmss}-计划摘要.md
```

同一任务只允许这一份文件，从询问、计划、验收、执行、打回到最终结论都更新它。

## 目录结构

```text
task-driver/
  SKILL.md
  README.md
  VERSION.md
  CHANGELOG.md
  references/
    quick-start.md
  scripts/
    check-contracts.sh
```

参考文件只保留极简快速入口。
