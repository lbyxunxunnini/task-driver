# 模板参考

SKILL.md 各环节引用的展示模板，集中维护于此。

## 轻量计划摘要模板

文件名：`.task-driver/{任务标题}-{YYYYMMDDHHmmss}-计划摘要.md`

```markdown
# {任务标题} 计划摘要

## 目标

## 已确认内容
| 项目 | 内容 | 状态 |
|---|---|---|

## 计划

### 1. {计划事项}
- 已确认：
- 实施方案：
- 影响范围：
- 状态：

## 验收目标
| 序号 | 最终验收目标 | 宿主任务 ID（如有） | 内部检查方式 | 状态 |
|---|---|---|---|---|

## 目标执行记录
| 轮次 | 验收目标 | 状态 | 打回原因 | 处理结果 |
|---|---|---|---|---|

## 当前结论
```

## 计划草案展示模板

```markdown
# 计划草案

任务目标：{本次任务最终要达成什么}

计划依据：
- 已确认 {N} 个需求或问题。
- 以下计划事项与它们逐项对应。

## 1. {计划事项}

### 已确认

- {讨论中确认的关键决策和边界}

### 预期变化

现状：
{当前问题或缺失能力}

实施后：
{用户或系统将表现成什么样}

### 实施方案

{采用的核心机制、流程变化、作用边界和关键约束}

### 涉及位置

- {主要文件、模块、流程或产物，以及准备调整什么}

### 内部验证

- {模型准备使用的行为回放、测试或检查}
```

## 统一异常通知骨架

```markdown
## {未达标 / 需要用户验收 / 需要用户授权或决定 / 受阻}

当前结果：{当前做到什么程度}

原因：{为什么没有继续或没有达标}

已完成：
- {已经完成的内容}

下一步：{agent 将怎样修复，或用户需要完成什么}
```

## 用户验收通知格式

```markdown
## 需要用户验收

当前结果：{agent 已完成的检查和当前产物状态}

原因：{agent 无法可靠验证的具体原因}

已完成：
- {已经完成的实现和内部验证}

验收步骤：
1. {具体操作}
2. {具体操作}
3. {具体操作}

通过条件：
- {用户应看到或得到的明确结果}

请反馈：
- {需要用户回复的通过/不通过结果，以及必要的截图、现象或其他证据}

下一步：收到反馈后，达标则完成目标；未达标则继续修复和复验。
```
