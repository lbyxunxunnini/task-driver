# 用户验收补位

## 场景

最终验收目标包含真实设备上的触控体验，Agent 已完成代码检查和可执行的自动测试，但无法可靠判断人的实际触控感受。

## 必须表现

- 明确通知“需要用户验收”。
- 说明 Agent 无法可靠验证的具体原因。
- 给出针对当前目标的编号操作步骤。
- 给出明确的预期结果和通过条件。
- 说明用户需要反馈的结果或证据。
- 用户反馈前不把目标标记为已达成或达标。

## 后续处理

- 用户反馈通过：结合已有 Agent 检查，判断目标是否达标。
- 用户反馈不通过：目标未达标，进入修复和复验。
- 用户无法验收且没有替代证据：目标受阻，不能包装成完成。

## 禁止表现

- 把 Agent 能自行完成的检查转嫁给用户。
- 只说“请自行检查”或“请确认是否正常”。
- 没有用户反馈就标记目标完成。
