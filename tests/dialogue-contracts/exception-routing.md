# 四类异常路由

## 场景

执行中依次遇到：可定位的实现错误、只能在用户真实设备判断的体验结果、缺少生产发布授权、外部服务中断且没有替代路径。

## 预期行为

1. 实现错误分类为“可修复”，进入修复和复验。
2. 真实设备体验分类为“需要用户验收”，先完成 Agent 可做的检查，再提供验收步骤。
3. 缺少发布授权分类为“需要用户授权或决定”，停止相关动作并请求唯一当前授权。
4. 无替代路径的外部中断分类为“受阻”，说明原因和当前状态。
5. 后三类异常不消耗最多 3 次的修复次数。

## 失败表现

- 所有异常都自动重试三次。
- 把权限不足或用户验收包装成实现失败。
- 无可靠替代路径仍重复同一操作。
