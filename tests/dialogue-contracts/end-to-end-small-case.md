# 端到端小案例

## 场景

用户需要解决两个问题：登录按钮在桌面端错位，以及在移动端窄屏下溢出。

## 对话路径

```text
用户要求逐项讨论两个问题
→ 模型一次只讨论一个问题
→ 用户逐项确认
→ 模型判断讨论充分并提供继续讨论或生成计划的选择
→ 用户选择生成计划
→ 计划包含两个对应事项，每项展示已确认决策和实施方案
→ 用户确认计划
→ 模型逐项讨论两个问题的最终完成状态
→ 模型提供继续讨论或生成最终验收目标的选择
→ 用户选择生成目标
→ 输出两个与原问题对应的必要终极目标
→ 用户确认后原样创建目标并执行
```

## 五项目标覆盖

1. 讨论未结束前不输出计划。
2. 计划确认后不直接输出验收目标。
3. 最终验收目标不扩散成施工检查清单。
4. 每个计划事项包含已确认决策和实施方案。
5. 两个问题、两个计划事项和两个最终目标保持对应。

## 失败判定

- 任一步提前输出下一阶段完整草案。
- 计划缺少实施方案或包含未经讨论的关键取舍。
- 最终目标出现文件、测试、脚本或版本更新等内部工作。
- 任一原始问题在计划或最终目标中失去对应关系。
