# 渐进式文档路由

## 场景

用户第一次接触任务驱动，只想判断是否适合当前任务；随后分别询问“三次修复是什么意思”和“跨模块重构应该重点讨论什么”。

## 预期行为

1. README 先让用户快速判断是否使用，并提供 Quick Start、FAQ、场景指南和完整协议入口。
2. 流程和示例只进入 Quick Start，不要求先阅读完整协议。
3. 重试问题从独立 FAQ 得到直接答案。
4. 跨模块重构从场景指南得到讨论、计划、验证、用户验收和停止边界。
5. 各文档只承担自己的职责，不大段复制完整协议。

## 失败表现

- README 一次性展开全部执行协议。
- FAQ 复制完整主流程而没有直接回答问题。
- 场景建议替代用户对范围和取舍的决定。
