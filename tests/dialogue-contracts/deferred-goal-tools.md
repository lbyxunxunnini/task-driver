# Deferred 目标工具与宿主状态同步

## 场景一：TaskCreate schema 尚未加载

用户已经确认最终验收目标，宿主提供 deferred `TaskCreate`，但当前上下文没有它的 schema。

## 预期行为

1. 模型不得猜测 `TaskCreate` 参数。
2. 模型先执行 `ToolSearch query="select:TaskCreate"`。
3. schema 加载成功后，模型按参数定义调用 `TaskCreate`。
4. 如果返回 `Invalid tool parameters`，模型重新加载或核对 schema 后只重试一次。
5. schema 加载后仍失败时，模型进入目标工具失败路径，不机械重试。

## 场景二：任务创建后的状态闭环

宿主使用任务列表承载多条最终验收目标，执行过程发生上下文压缩后继续。

## 预期行为

1. 每条最终验收目标只对应一条宿主任务，实施步骤不创建为宿主任务。
2. 创建前通过 `TaskList` 查询已有任务，存在对应任务时直接复用。
3. 创建后保存任务 ID，开始执行时通过 `TaskUpdate` 更新为执行中。
4. 只有最终验收目标确认达标后，才通过 `TaskUpdate` 更新为已完成。
5. 中断恢复时优先按任务 ID 恢复映射，没有 ID 时才按最终验收目标原文匹配。
6. 发现重复任务时先识别有效项，不批量伪标为完成，也不再次创建。

## 失败表现

- schema 未加载时直接调用 `TaskCreate`。
- 使用相同的猜测参数连续重试。
- 把文件修改、测试步骤或施工清单创建为宿主任务。
- 工作完成后只更新内部状态，不调用 `TaskUpdate`。
- 中断恢复或调用失败后重复创建相同任务。
- 为了清理重复项，把未经最终验收的任务批量标记为已完成。
